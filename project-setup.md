# Project Setup

## Quick start

```bash
mkdir myproject && cd myproject
zig init
```

This creates:
```
myproject/
├── build.zig        # Build configuration
├── build.zig.zon    # Package manifest
└── src/
    ├── main.zig     # Entry point
    └── root.zig     # Library root
```

## build.zig

This is Zig's build system — it replaces Makefiles, CMake, etc. It's **written in Zig itself**.

```zig
const std = @import("std");

pub fn build(b: *std.Build) void {
    // Let the user choose target (cross-compilation) and optimization
    const target = b.standardTargetOptions(.{});
    const optimize = b.standardOptimizeOption(.{});

    // Define the executable
    const exe = b.addExecutable(.{
        .name = "myapp",
        .root_source_file = b.path("src/main.zig"),
        .target = target,
        .optimize = optimize,
    });

    // Install the binary to zig-out/bin/
    b.installArtifact(exe);

    // "zig build run" step
    const run_cmd = b.addRunArtifact(exe);
    run_cmd.step.dependOn(b.getInstallStep());
    if (b.args) |args| {
        run_cmd.addArgs(args);
    }
    const run_step = b.step("run", "Run the application");
    run_step.dependOn(&run_cmd.step);
}
```

## build.zig.zon

Package manifest. In Zig 0.14+, uses this format:

```zig
.{
    .name = .myapp,              // Enum literal, NOT a string
    .version = "0.1.0",
    .fingerprint = 0x...,        // Auto-generated, don't invent one
    .minimum_zig_version = "0.14.0",
    .dependencies = .{},
    .paths = .{
        "build.zig",
        "build.zig.zon",
        "src",
    },
}
```

**Important**: The `.name` field is an enum literal (`.myapp`), not a string (`"myapp"`). If you need special characters, use `@""` syntax: `.name = .@"my-app"`.

The `.fingerprint` field is generated by Zig. If you make one up, the build will fail and tell you the correct value. Just copy what it suggests.

## Build commands

```bash
zig build              # Build (output in zig-out/bin/)
zig build run          # Build and run
zig build run -- arg1  # Build and run with arguments
zig build -Doptimize=ReleaseFast  # Optimized build
```

## Adding C source files

Zig can compile C code directly — no separate C compiler needed:

```zig
// In build.zig
exe.addCSourceFile(.{
    .file = b.path("deps/sqlite3.c"),
    .flags = &.{"-DSQLITE_THREADSAFE=1"},
});
exe.addIncludePath(b.path("deps"));
exe.linkLibC();  // Link libc (needed by most C code)
```

## Directory conventions

```
project/
├── build.zig          # Build config
├── build.zig.zon      # Package manifest
├── src/               # Zig source
│   ├── main.zig       # Entry point (pub fn main)
│   └── *.zig          # Other modules
├── deps/              # C dependencies (if any)
└── zig-out/           # Build output (gitignore this)
    └── bin/
        └── myapp
```

Add to `.gitignore`:
```
zig-out/
.zig-cache/
```
