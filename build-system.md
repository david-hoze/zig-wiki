# Build System

Zig's build system is written in Zig itself. The `build.zig` file replaces Makefiles, CMake, etc.

## Basic build.zig

```zig
const std = @import("std");

pub fn build(b: *std.Build) void {
    const target = b.standardTargetOptions(.{});
    const optimize = b.standardOptimizeOption(.{});

    const exe = b.addExecutable(.{
        .name = "myapp",
        .root_source_file = b.path("src/main.zig"),
        .target = target,
        .optimize = optimize,
    });

    b.installArtifact(exe);
}
```

## Build commands

```bash
zig build                         # Build (output: zig-out/bin/myapp)
zig build -Doptimize=Debug        # Debug build (default)
zig build -Doptimize=ReleaseFast  # Optimized for speed
zig build -Doptimize=ReleaseSmall # Optimized for size
zig build -Doptimize=ReleaseSafe  # Optimized but keeps safety checks
```

## Run step

```zig
const run_cmd = b.addRunArtifact(exe);
run_cmd.step.dependOn(b.getInstallStep());
if (b.args) |args| {
    run_cmd.addArgs(args);
}
const run_step = b.step("run", "Run the application");
run_step.dependOn(&run_cmd.step);
```

```bash
zig build run                    # Build and run
zig build run -- arg1 arg2       # Build and run with arguments
```

## Including C sources

```zig
// Single C file
exe.addCSourceFile(.{
    .file = b.path("deps/sqlite3.c"),
    .flags = &.{
        "-DSQLITE_THREADSAFE=1",
        "-DSQLITE_ENABLE_WAL=1",
        "-std=c99",
    },
});

// Tell the C compiler where to find headers
exe.addIncludePath(b.path("deps"));

// Link libc (needed by most C code)
exe.linkLibC();
```

## Multiple C files

```zig
const c_flags = &.{ "-std=c99", "-O2" };

exe.addCSourceFile(.{ .file = b.path("deps/file1.c"), .flags = c_flags });
exe.addCSourceFile(.{ .file = b.path("deps/file2.c"), .flags = c_flags });
```

## Linking system libraries

```zig
exe.linkSystemLibrary("pthread");
exe.linkSystemLibrary("z");  // zlib
```

## build.zig.zon (Zig 0.14+)

```zig
.{
    .name = .myapp,                    // Enum literal, not string
    .version = "0.1.0",
    .fingerprint = 0x...,             // Auto-generated by zig
    .minimum_zig_version = "0.14.0",
    .dependencies = .{},
    .paths = .{
        "build.zig",
        "build.zig.zon",
        "src",
        "deps",
    },
}
```

### Fingerprint

If you make up a fingerprint, the build fails and tells you the correct one:
```
error: invalid fingerprint: 0xDEAD; use this value: 0x21ad9112faecb50e
```
Just copy what it suggests.

### Name format

The `.name` must be an enum literal:
- `.name = .myapp` (simple)
- `.name = .@"my-app"` (with special characters)
- NOT `.name = "myapp"` (this is a string, not an enum literal)

## Cross-compilation

Zig can cross-compile to any target without extra toolchains:

```bash
# Build for Linux ARM from any platform
zig build -Dtarget=aarch64-linux-gnu

# Build for Windows from Linux
zig build -Dtarget=x86_64-windows

# Build for macOS from Linux
zig build -Dtarget=x86_64-macos
```

This works because Zig bundles its own libc and linker for all targets.

## Output location

```
project/
├── zig-out/           # Build output
│   └── bin/
│       └── myapp      # The executable
└── .zig-cache/        # Build cache (speeds up rebuilds)
```

Both should be in `.gitignore`.

## Test step

```zig
const tests = b.addTest(.{
    .root_source_file = b.path("src/main.zig"),
    .target = target,
    .optimize = optimize,
});

const test_step = b.step("test", "Run unit tests");
test_step.dependOn(&b.addRunArtifact(tests).step);
```

```bash
zig build test
```
